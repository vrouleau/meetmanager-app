# Overview

**Meet Manager** is a web application for managing lifesaving competition registrations. Clubs log in with a PIN, register their athletes for events, and the organizer oversees the meet setup, invitations, and export. An admin manages the overall platform.

The application works alongside **SPLASH Meet Manager 11**: the meet structure is exported from SPLASH and uploaded to Meet Manager App, and the final registrations are exported back into SPLASH before race day.

## Roles

| Role | Description |
|------|-------------|
| **Admin** | Full access. Manages clubs, athletes, app configuration, organizer designation, data management, and post-meet results upload. |
| **Organizer** | A club designated by the Admin. Uploads the meet structure, sets the closure date, sends invitations, manages Stripe billing, and exports registrations. Also has access to Athletes. |
| **Coach** | A club login. Views and manages their own athletes and registrations. |

The Admin PIN is set in the `.env` file (default: `314159`) and can be changed in the Admin page. Each club has a 6-digit PIN generated by the system. An organizer logs in with their club PIN and gains Organizer access because their club is flagged as the organizer.

## Navigation

After login the navigation bar shows links based on your role:

- **Athletes** — visible to all roles
- **Organizer** — visible to Admin and Organizer
- **Admin** — visible to Admin only
- **Data Management** — visible to Admin only

The current meet name is shown in the navigation bar when a meet is loaded. Language can be toggled (FR / EN) from any page.

# Admin Guide

## Logging In as Admin

Enter the admin PIN on the login page. The default PIN is `314159`. After login the role is `admin`.

## Status Overview

The top of the Admin page shows a live count of: clubs, athletes, events, registrations, and best times in the database.

## Uploading Clubs and Athletes (Lenex)

The **Upload Lenex (.lxf)** section accepts a Lenex-format `.lxf` file (entries or results export from SPLASH). On upload:

- A preview shows how many clubs and athletes are in the file and how many are new
- Confirm to proceed — existing clubs and athletes are matched by club code and NRAN license, not overwritten
- New best times are imported and merged (fastest time per style per pool size is kept)
- Each best time is stamped with the date of the source meet. Times older than 18 months are automatically purged when an athlete's registration page is opened.

Use this to seed the app before the first meet of the season, or after each meet to import results.

## Managing Clubs

The **Club Manager** table lists all clubs with their PIN, email, and athlete count.

- **Edit email**: click the email field and edit in-place; changes save on blur
- **Delete club**: removes the club along with all its athletes, registrations, and best times (confirmation required)
- **Add club**: enter a name and optional email, then click Add — a 6-digit PIN is generated automatically

## Regenerating All Club PINs

The **Regenerate All Club PINs** button generates new 6-digit PINs for every club. Use this when you want to revoke all existing logins (e.g., between seasons). Coaches will need to receive their new PINs via the invitation flow.

## Changing the Admin PIN

Enter a new PIN (minimum 4 characters) in the **Change Admin PIN** form and submit. The new PIN takes effect immediately. Your session cookie is updated automatically.

## Designating the Organizer

The **Set Meet Organizer** section lets you assign a club as the organizer for the current meet. The designated club's coach will see the **Organizer** page in addition to **Athletes**. Only one organizer is active at a time. The designation is cleared when the meet is flushed.

## Flushing the Meet

The **Flush Meet** button deletes:

- All registrations
- All events (meet structure)
- The organizer designation
- Meet metadata (name, dates, fees, closure date)

Clubs, athletes, best times, and PINs are **not** deleted. Use Flush Meet to reset the app between competitions while keeping the athlete database intact.

## Uploading Results (Post-Meet)

After the competition, export results from SPLASH and upload the results `.lxf` in the **Upload Lenex (.lxf)** section. The app imports results and updates best times: for each athlete and each event style, the faster of the existing best time and the new result is kept, separately for LCM (50m) and SCM (25m).

# Organizer Guide

## Accessing the Organizer Page

Log in with your club PIN. Because your club has been designated as organizer by the Admin, you will see both **Athletes** and **Organizer** in the navigation.

## Meet Information Banner

The top of the Organizer page shows the loaded meet: name, pool size (50m / 25m), Masters flag, event count, filename, and upload date. If no meet is loaded, a warning is shown.

## Fee Summary

The **Fee Summary** box (scrollable, monospace) shows:

- **Meet-level fees**: per club, per athlete, per relay team, per team, late entry fee, LSC fee
- **Per-entry fees by event**: event number, style name, and fee amount

These fees are read from the meet `.lxf` structure. They are used by the invoice generation to calculate each club's total.

## Downloading the Meet Template

Click **Download Meet Template (.lxf)** to download the stored base meet file. Open this file in SPLASH to restore the previous meet structure as a starting point. Customize the events, sessions, dates, and fees in SPLASH, then export the invitation `.lxf` and upload it back.

## Uploading the Meet Structure

Click **Upload Meet Structure (.lxf)** and select the invitation `.lxf` exported from SPLASH. This loads the event structure into the app. If a meet is already loaded, a warning explains that all current registrations, the closure date, and club PINs will be reset — confirm to proceed.

## Setting the Entry Closure Date

Click the date field in the **Entry closure date** row and select the last date for entries. Save by clicking outside the field. After the closure date passes:

- Registrations are no longer accepted from coaches
- The invitation table greys out (invitations can still technically be sent, but entries are closed)
- Invoice buttons become active (Stripe invoices can only be sent after closure)

## Sending Invitations

The **Team Invites** table lists all clubs with their email and invoice status.

- **Send to selected**: check the clubs you want to invite and click **Send Invitation (n)**
- Each coach receives an email with a one-time secure link. The link reveals their club PIN when clicked. The link expires after 48 hours and can only be used once.

> Clubs without an email address cannot receive invitations. Set the email in the Admin → Club Manager.

## Exporting Registrations

Click **Download bundle (.zip)** to export the registrations. The zip contains:

- `inscriptions_bundle.zip` (outer container)
  - A Lenex `.lxf` file with all registrations, athletes, clubs, and entry times
  - `simulate_results.bat` — a Windows launcher for the simulation script
  - `simulate_results.vbs` — a VBScript for simulating results in SPLASH on meet day

Import the `.lxf` into SPLASH via **Transfers → Import entries…** to load all athletes and entry times for race day.

## Stripe Billing

### Connecting Stripe

Click **Connect Stripe** to start the Stripe Connect OAuth flow. You will be redirected to Stripe to authorize the connection. Once connected, the button changes to show a green "Connected" status with a Disconnect option.

### Sending Invoices

After the closure date, each club row in the **Team Invites** table shows a **Send Invoice (amount)** button if:

- The club has at least one registration (invoice total > 0)
- Your Stripe account is connected

Click to create and send a Stripe invoice to that club via your connected Stripe account. The invoice is calculated from the fee structure in the meet `.lxf`.

### Downloading PDF Invoices (Without Stripe)

If Stripe is not connected, a **Download Invoice** button appears instead. This generates a PDF invoice (via reportlab on the server) that you can download and send manually.

### Invoice Calculation

Each invoice includes:

| Line item | Rule |
|-----------|------|
| Per club | 1 × club fee |
| Per athlete | Number of athletes with at least 1 registration × athlete fee |
| Per relay | Number of distinct relay events entered × relay fee |
| Per team | 1 × team fee (if declared) |
| Late fee / LSC fee | 1 × fee (if declared) |
| Per individual event | 1 per athlete registered in that event × event fee |
| Per relay event | 1 per team in that relay event × relay event fee |

Items with a zero fee or zero quantity are omitted from the invoice.

# Club Coach Guide

## Logging In

Enter your club PIN on the login page. You will land on the **Athletes** page showing only your club's athletes.

> If you have not received your PIN yet, ask the meet organizer to send you an invitation email. Click the link in the email to reveal your PIN.

## Secret PIN Links

When the organizer sends an invitation, you receive an email with a one-time link. Clicking the link shows your club PIN on a secure page. The link expires after 48 hours and can only be used once. Store your PIN safely — you will need it to log in.

## Viewing Athletes

The **Athletes** page lists all athletes in your club with name, gender, birthdate, and license number. Use the search box to filter by name.

### Adding an Athlete

Click **+ Add Athlete**, fill in first name, last name, gender, birthdate, and NRAN license, then save.

### Editing an Athlete

Click **Edit** next to the athlete's name to update their details.

### Deleting an Athlete

Click **Delete** next to an athlete's name. This removes the athlete and all their registrations and best times (confirmation required).

## Registering an Athlete

Click on an athlete's name to open the **Registration** page for that athlete.

### Individual Events

The **Individual Events** section lists all available events grouped by gender match. For each event:

- **Category** — age group automatically suggested based on the athlete's age and the meet year (10-, 11-12, 13-14, 15-18, Open). Masters is never auto-suggested.
- **Best Time 50m / Best Time 25m** — displayed read-only from the athlete's stored best times. Times older than 18 months are considered expired and will not appear.
- **Entry Time** — pre-filled from the best time matching the meet pool size; can be edited
- Check the event checkbox to register; uncheck to deregister

### Relay Events

The **Relays** section lists relay events. A club can field one relay team per relay style. If another athlete from your club is already registered for a relay, the event is locked for additional athletes with a note showing who registered first.

### Age Code

Age is computed against December 31 of the meet year. The app automatically suggests the correct age group. You can change the category from the dropdown if needed (e.g., for Open category).

# Data Management (Admin)

Access the **Data Management** page from the navigation bar (Admin only).

## Club Merging

If the same club appears under different names (e.g., due to import variations across meets), use the **Club Merging** table to map duplicates to their canonical entry.

- Each row shows a club name in the **From** column and a dropdown in the **To** column
- Set the **To** club to the canonical version for any duplicate; rows with a pending merge are highlighted yellow
- Click **Resolve** to merge — all athletes and registrations from the "From" club are moved to the "To" club, and the "From" club is deleted
- **This action cannot be undone**

## Event Style Merging

If best times reference different style UIDs for the same event (possible when importing from different SPLASH databases), use **Event Style Merging** to consolidate them.

- Map diverging style IDs to the canonical one; pending changes are highlighted yellow
- Click **Resolve** — best times are merged, keeping the faster time per pool size

## Exporting All Data (Entries)

Click **Download entries (.lxf)** to export a Lenex file containing all clubs, athletes, and best times. This is equivalent to SPLASH's "Export entries" function. Save this file to use as the seed for the next meet (upload it in the Admin page at the start of the season).

# Season Lifecycle

## Starting a New Season

1. **Export all data** from Data Management → Download entries (.lxf) — save as your seed file
2. Run **Flush Meet** in the Admin page to clear the current meet
3. Upload the seed file (from step 1) back via Admin → Upload Lenex to restore clubs, athletes, and best times

Alternatively, export the entries `.lxf` directly from SPLASH after importing the current season's results, then upload it to seed the new season.

## Starting a New Meet (Mid-Season)

If you are running multiple meets per season and want to clear registrations without resetting clubs:

1. Run **Flush Meet** — this deletes registrations, events, closure date, and organizer designation
2. The organizer repeats the meet setup flow (Steps 2–6 in the Quick Start Workflow)

## Resetting All Club PINs

Use **Regenerate All Club PINs** in the Admin page. All coaches will need to receive new PINs via the invitation flow. This is useful between seasons when new coaches may have taken over club accounts.

# Troubleshooting

## Athlete Not Appearing in Registration

- Confirm the meet has been uploaded (green banner in Organizer page)
- Check the athlete's gender matches the event gender
- Check the closure date has not passed (entries are locked after closure)

## Entry Time Not Pre-Filling

The entry time pre-fills from the athlete's best time for the pool size matching the current meet (LCM = 50m, SCM = 25m). If no best time exists for that pool size, the field will be blank. Enter the time manually.

## Invitation Email Not Received

- Confirm the club's email is set correctly in Admin → Club Manager
- Ask the organizer to re-send the invitation
- Check the spam/junk folder
- PIN links expire after 48 hours — a new invitation must be sent if the link has expired

## Stripe Invoice Error

- Confirm the Stripe account is connected (green indicator in Organizer page)
- The club must have a valid email address
- Invoice buttons are only active after the closure date has passed

## Cannot Upload Meet Structure

- The file must be a valid Lenex `.lxf` exported from SPLASH via Transfers → Export invitation
- If a meet is already loaded, confirm the replacement warning
